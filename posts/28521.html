Oracle11gR2 Centos6.x静默安装

发表于 2017-12-14 更新于 2017-12-17 分类于 oracle 阅读次数：

本文资料来源：
Oracle® Database Quick Installation Guide（linux_x64）
Oracle® Database Quick Installation Guide（linux_x86）

一、下载

下载地址：Oracle Database Software Downloads


第一张图勾选圈出的选项同意许可证协议、第二张图选择正确的版本下载（注意是11gR2的Linux版本）、

二、准备

（一）内存空间

安装Oracle11gR2实例必须满足：
最小： 1 GB内存
建议： 2GB内存或更多

查询内存命令：grep MemTotal /proc/meminfo 也可以用free或top

建议用以下表格关系来根据内存大小配置交换空间大小

有效内存 交换分区
1GB到2GB之间 1.5倍内存大小
2GB到16GB之间 等于内存大小
超过16GB 16 GB

（一）磁盘空间

安装分区5G以上（不包含数据文件）、详情参照表格、/tmp文件夹至少1GB空闲空间

安装版本 软件大小 数据大小
Enterprise Edition 4.7 GB 1.7 GB
Standard Edition 4.6 GB 1.5 GB

三、安装

  1. 依赖包安装 —— root用户操作（以下安装包为个人整理）

    • Centos6.x 32位所需安装包：

      # 32位系统只会安装i686包
yum -y install binutils compat-libcap1 compat-libstdc++-33 gcc gcc-c++ \
glibc glibc-devel ksh libgcc libstdc++ libstdc++-devel libaio \
libaio-devel make sysstat unixODBC unixODBC-devel elfutils-libelf-devel

    • Centos6.x 64位所需安装包： 

      # 64位系统需要x86_64包
yum -y install gcc.x86_64 make.x86_64 binutils.x86_64 glibc.x86_64 \
compat-libstdc++-33.x86_64 elfutils-libelf.x86_64 \
elfutils-libelf-devel.x86_64 glibc-common.x86_64 glibc-devel.x86_64 \
glibc-headers.x86_64 gcc-c++.x86_64 libaio-devel.x86_64 libgcc.x86_64 \
libstdc++.x86_64 libstdc++-devel.x86_64 sysstat.x86_64 unixODBC.x86_64 \
unixODBC-devel.x86_64 expat.x86_64 ksh.x86_64
      # 64位系统需要i686包
yum -y install libaio.i686 glibc.i686 compat-libstdc++-33.i686 \
elfutils-libelf.i686 elfutils-libelf-devel.i686 libaio-devel.i686 \
libgcc.i686 libstdc++.i686 unixODBC.i686 unixODBC-devel.i686

  1. Oracle用户创建 —— root用户操作

    • 创建Oracle 安装用户组：/usr/sbin/groupadd oinstall

    • 创建Oracle DBA用户组：/usr/sbin/groupadd dba

    • 创建Oracle 用户：/usr/sbin/useradd -g oinstall -G dba oracle

    • 设置Oracle 用户密码：passwd oracle

  1. 系统参数配置 —— root用户操作

    • 打开vi /etc/sysctl.conf添加以下参数配置（官网给的最小配置）：

       fs.aio-max-nr = 1048576
 fs.file-max = 6815744
 kernel.shmall = 2097152
 kernel.shmmax = 536870912
 kernel.shmmni = 4096
 kernel.sem = 250 32000 100 128
 net.ipv4.ip_local_port_range = 9000 65500
 net.core.rmem_default = 262144
 net.core.rmem_max = 4194304
 net.core.wmem_default = 262144
 net.core.wmem_max = 1048576

    • 修改完成后执行/sbin/sysctl -p使配置立即生效

  1. Oracle用户限制 —— root用户操作

    • 打开vi /etc/security/limits.conf添加以下参数配置（官网给的最小配置）：

      oracle          soft    nofile          1024
oracle          hard    nofile          65536
oracle          soft    nproc           2047
oracle          hard    nproc           16384
oracle          soft    stack           10240
oracle          hard    stack           32768

    • 修改完成后重新登录oracle用户查看是否生效（参考官网-如下图：）

  1. 创建安装目录与权限 —— root用户操作

    • 创建安装目录：mkdir -p /u01/app/

    • 修改安装目录所属用户和组：chown -R oracle:oinstall /u01/app/

    • 修改安装目录权限：chmod -R 775 /u01/app/

  1. 安装Oracle数据库软件 —— oracle用户操作

    • 解压Oracle数据库软件压缩文件unzip ~/linux.x64_11gR2_database_1of2.zip && unzip ~/linux.x64_11gR2_database_1of2.zip

    • 打开vi ~/.bash_profile添加以下安装所需环境变量：

       ORACLE_BASE=/u01/app/oracle
 ORACLE_SID=orcl
 export ORACLE_BASE ORACLE_SID

    • 编辑静默安装模板文件（软件相关配置）、模板文件在解压后的目录下database/response/db_install.rsp、模板文件的详解参考其他的博客吧、这里不再详细说明

    • 执行安装 ~/database/runInstaller -silent -force -responseFile ~/database/response/db_install.rsp

  2. 执行清单权限相关脚本 —— root用户操作

    • 运行 /u01/app/oraInventory/orainstRoot.sh && /u01/app/oracle/product/11.2.0/dbhome_1/root.sh

      orainstRoot.sh：设置产品目录位置与拥有该目录的操作系统组、并产生/etc/oraInst.loc
      root.sh：用来设置必要的操作系统权限、并将orahome、oraenv、coraenv复制到/usr/local/bin目录下、生成/etc/oratab、其中存放一些数据库相关信息

  1. 配置Oracle数据库 —— oracle用户操作

    • 打开vi ~/.bash_profile添加以下软件所需环境变量：

      ORACLE_HOME=$ORACLE_BASE/product/11.2.0/dbhome_1
TNS_ADMIN=$ORACLE_HOME/network/admin
PATH=$PATH:$ORACLE_HOME/bin
LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$ORACLE_HOME/lib:/lib:/usr/lib:/usr/local/lib
CLASSPATH=$CLASSPATH:$ORACLE_HOME/jlib:$ORACLE_HOME/rdbms/jlib:$ORACLE_HOME/network/jlib:$ORACLE_HOME/lib:$ORACLE_HOME/ctx/lib
ORACLE_OWNER=oracle
SPFILE_PATH=$ORACLE_HOME/dbs
ORA_NLS10=$ORACLE_HOME/nls/data
export ORACLE_HOME TNS_ADMIN PATH LD_LIBRARY_PATH CLASSPATH ORACLE_OWNER SPFILE_PATH ORA_NLS10

    • 配置监听 $ORACLE_HOME/bin/netca /silent /responseFile ~/database/response/netca.rsp

    • 编辑数据库实例模板文件（实例相关配置）、模板文件在解压后的目录下database/response/dbca.rsp、模板文件的详解参考其他的博客吧、这里不再详细说明

    • 创建实例 $ORACLE_HOME/bin/dbca -silent -responseFile ~/database/response/dbca.rsp

四、配置

（一）自启动

  1. 修改vi /etc/oratab —— root用户操作
    # This file is used by ORACLE utilities.  It is created by root.sh
# and updated by the Database Configuration Assistant when creating
# a database.
 #
# A colon, ':', is used as the field terminator.  A new line terminates
# the entry.  Lines beginning with a pound sign, '#', are comments.
#
# Entries are of the form:
#   $ORACLE_SID:$ORACLE_HOME:<N|Y>:
#
# The first and second fields are the system identifier and home
# directory of the database respectively.  The third filed indicates
# to the dbstart utility that the database should , "Y", or should not,
# "N", be brought up at system boot time.
#
# Multiple entries with the same $ORACLE_SID are not allowed.
#
# 改变此处N为Y
MFGS:/u01/app/oracle/product/11.2.0/dbhome_1:Y

  1. 配置自启动脚本 —— root用户操作

    #!/bin/sh
#chkconfig: 345 91 91
#description: oracle autorun services
#/etc/init.d/oracle
#
#Run-level Stratup script for the Oracle Instance, Listener, and Web Interface

AUTO_FILE_NAME=oracle
export ORACLE_BASE=/u01/app/oracle
export ORACLE_HOME=$ORACLE_BASE/product/11.2.0/dbhome_1
export ORACLE_SID=orcl
export PATH=$PATH:$ORACLE_HOME/bin

ORA_OWNR="oracle"

case "$1" in
 start)
     su $ORA_OWNR -lc $ORACLE_HOME/bin/dbstart
     touch /var/lock/subsys/$AUTO_FILE_NAME
     echo "Oracle Start Successful!"
     ;;
 stop)
     su $ORA_OWNR -lc $ORACLE_HOME/bin/dbshut
   rm -f /var/lock/subsys/$AUTO_FILE_NAME
     echo "Oracle Stop Successful!"
     ;;
 reload|restart)
     $0 stop
     $0 start
     ;;
 *)
     echo "Usage: `basename $0` {start|stop|reload|restart}"
     exit 1
esac
exit 0

#chmod 755 /etc/init.d/oracle
#chkconfig --add oracle
#chkconfig --list oracle
